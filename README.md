# 코스피 야간선물 GitHub Pages · Notion 차트

Hang on! 페이지가 서버에서 내려주는 코스피 야간선물 원본 데이터에서 현재 지수, 등락폭, 등락률과 전체 1,000개 시계열을 직접 읽어 GitHub Pages로 배포합니다.

## 이번 오류의 원인과 수정

이전 파일은 브라우저 차트의 네트워크 응답과 툴팁을 읽으려고 했지만, 실제 원본 데이터는 Next.js HTML 내부에 이스케이프된 JSON으로 들어 있었습니다. 그 결과 `원본 전체 차트 시계열을 찾지 못했습니다` 오류가 발생했습니다.

수정 파일은 브라우저나 툴팁을 사용하지 않고 HTML 안의 원본 `history` 배열을 직접 읽습니다. 현재 원본 기준으로 시작 시각부터 종료 시각까지 정확히 1,000개 포인트를 가져옵니다. 원본이 일시적으로 응답하지 않더라도 워크플로를 실패시키지 않고 직전 정상 차트를 유지합니다.

## 반드시 교체할 파일

다음 세 파일을 모두 덮어써야 합니다.

1. 저장소 루트의 `update.py`
2. 저장소 루트의 `requirements.txt`
3. `.github/workflows/update.yml`

특히 세 번째 파일이 중요합니다. 오류 화면의 작업 이름이 `코스피 야간선물 패널 가져오기`로 표시되면 예전 워크플로가 남아 있는 것입니다. 수정 후에는 작업 이름이 `코스피 야간선물 원본 데이터 가져오기`로 표시되어야 합니다.

## 적용 방법

1. ZIP을 풉니다.
2. GitHub의 `kospi-night-futures-dashboard` 저장소에 이 폴더 안의 파일을 같은 위치로 덮어씁니다.
3. GitHub에서 `.github/workflows/update.yml` 파일까지 변경되었는지 직접 확인합니다.
4. **Settings → Pages → Build and deployment → Source**를 **GitHub Actions**로 선택합니다.
5. **Actions → 코스피 야간선물 자동 갱신 → Run workflow**를 실행합니다.
6. 로그에 `원본 차트 전체 구간`과 `(1000개)`가 나오면 정상입니다.
